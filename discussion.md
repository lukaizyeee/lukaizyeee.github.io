## 1. 定位与受众

### 1.1 网站定位

- **综合门户 + 长期维护的个人展示面**：不是求职简历的网页版
- **求职简历是独立的、更正式的 PDF**，用于投递；网站是承载"更完整的我"的地方
- **允许有成长轨迹**：作者仍是学生，网站要能体现思考、学习、方向感，而不是伪装成"资深作品集"

### 1.2 主要受众（按优先级）

1. **未来学术同行、潜在项目合作者 / IPADS 方向的老师、学长学姐**（首要）
   - 看到网站时，第一印象要与"操作系统 / AI Infra / Agent"这几个方向"对味"
2. **HR / 面试官**：会顺路看，但不是核心受众（他们主要看 PDF 简历）

### 1.3 内容基调

- **技术基调靠向 IPADS 方向**：操作系统、AI Infra、Agent、大模型系统工程
- 现有项目（合同审查系统）本身就属于"AI 系统工程"叙事，天然契合
- 需要在**方向表达 / 兴趣声明 / 学习记录**等模块显性地朝这些关键词收敛

### 1.4 核心目标（一句话）
>
> 让访问者在 30 秒内明白："这是一个正在系统性走向 AI 系统 / OS × AI 方向的本科生，他在做真项目、有工程品味、正在往 IPADS 靠。"

### 1.5 设计范围（用户明确）

- 本阶段目标：**搭建整体框架 + 确定视觉方案**，不在此阶段抠具体内容细节。
- 文案、博客文章、case study 正文等以后持续填充维护；设计产出以「信息架构 + 设计系统 + 页面模板」为准，讨论 demo 中的文案均为占位示意。
- 技术选型相应地要优化「后续加内容的成本」：加一篇文章/一个项目应该只是加一个 Markdown 文件。

## 2. 内容结构与信息架构

### 2.1 站点模块（多页面结构）

| 模块 | 路径（暂定） | 说明 |
|---|---|---|
| 首页 | `/` | 首屏方向声明 + 精选项目 + 最近文章入口 |
| 项目 | `/projects` + `/projects/<slug>` | 列表页 + 深度 case study 详情页 |
| 博客 / 技术笔记 | `/blog` + `/blog/<slug>` | 学习记录、读论文/读书笔记、方向思考 |
| Now | `/now` | 「我现在在做什么 / 学什么」，低成本高信号，定期手动更新 |
| 关于 & 联系 | `/about` | 完整自我介绍、联系方式、**PDF 简历下载链接** |

- **不做**独立网页版简历页：正式简历保持 PDF 形态，网站只提供下载入口（关于页）。
- **不做**兴趣/非技术板块（首版）：保持「AI 系统方向本科生」的第一印象聚焦；未来若想加，作为博客的一个分类即可。

### 2.2 语言策略

- **双语站，默认落地英文**，导航栏提供中/英切换。
- **覆盖范围**：固定页面（首页、关于、Now、项目 case study）维护中英两版；**博客文章单语**——每篇用什么语言写就用什么语言发布，不强制翻译。
- 技术术语（RAG、prompt、focused rerun 等）在两个语言版本中均保留英文原文。

### 2.3 首页首屏（30 秒目标的落点）

- **方向声明型**首屏：姓名 + 一句方向声明（AI systems / OS × AI 方向的工程实践者）+ 上海交通大学本科生身份 + 主要入口（Projects / Blog / Now）。

每个核心项目一个详情页，内容骨架：

1. **背景与动机**——解决什么问题、为什么做

## 3. 视觉风格与调性

### 3.1 已确认的审美偏好（来自参考站点反馈）

- **总体**：简约画面 + 有交互感和动感；喜欢深紫、米色、黑色。
- **底色**：Anthropic 官网的米色底让人舒适（正面参照）。
- **动效品味**：Apple 系统级的丝滑感。
  - ✅ Linear 首页加载时「从上到下缓缓浮现」的级联进场
  - ✅ rauno.me 的入场中心放大动画、横向滚动叙事、进度指示游标
  - ⚠️ rauno.me 的不足：缺少悬停反馈，模块交互感略欠 → **我们要补上悬停微交互**
  - ❌ Reflect 式的华丽发光：好看但太炫，不符合学术气质
- **气质边界**：视觉可以有工艺感和新奇感，但不能牺牲「学术 / 研究者」的可信度。
- **紫色倾向**：
  - 色板测试中选中 **A #5E6AD2（蓝紫）** 和 **C #5B21B6（深堇紫）**，两者都可接受。
  - 补充参照（Anthropic 官网截图两张）：**奶油米底 + 灰调薰衣草紫（≈#CAC7DE）大色块 + 近黑墨色文字**的淡雅搭配「很舒服」；中紫（≈#8578C7）做插画/封面色块也认可。
  - → 结论方向：紫色系统分层使用——淡薰衣草做大面积表面色，中高饱和紫做交互/点缀色。
- **色彩系统选定（方案 1）**：
  - 奶油米底 `#F7F4EE` / 淡薰衣草大色块 `#CDCAE0` / 中紫（插画、次级色块）`#8A82C9` / 交互紫（链接、按钮、高亮）`#5E6AD2` / 墨色文字 `#1C1917` / 白色卡片
  - 中性色阶（次级文字、边框、分隔线）从墨色与米色派生，不引入新色相。
- **强调手法（已确认）**：不完全依赖颜色强调——用黑色线条系统做强调与结构，与 Anthropic 的 ink-line 插画语言同源。6 种用法按喜好分层：
  - **签名元素（最喜欢，形成个人辨识度）**：① 细框按钮（悬停填充墨色+紫色硬投影）、③ 描边卡片+硬投影（悬停错位）、⑤ 手绘圈注（SVG 描边动画圈关键词，**用户最强偏好**）
  - **实用配角（简单但实用，辅助使用）**：② 动态下划线（导航/正文链接）、④ 编号+标尺线（章节标题）、⑥ 四角括号框（Now/数据高亮）
  - **手绘圈注实现要点**（讨论中踩坑）：`stroke-dasharray/-offset` 必须运行时用 `getTotalLength()` 精确设置，硬编码估算长度会在动画前泄漏一段笔画、起笔位置错乱；CSS 里保留一个偏大的兜底值防脚本执行前闪现。
  - **手绘圈注交互（已确认）**：进场自动画一次（延迟约 1.15s，级联进场之后）+ **每次鼠标悬停重画一次**；笔色暂定墨黑 `#1C1917`（对比过淡墨 45%/中紫/交互紫，改色零成本，留到最后微调）。
- **颜色数量结论**：5 个色相封顶（米/薰衣草/中紫/交互紫/墨）+ 派生中性阶 + 黑线条系统，不再加新色相；功能色（错误/成功）未来按需补充。
- **字体系统（已确认，A+C 混搭）**：
  - **标题**：衬线体（英文 Source Serif 类 / 中文思源宋体）——学术书卷气，Anthropic 同款策略
  - **正文**：无衬线（英文 Inter 类 / 中文思源黑体）——长文可读性
  - **等宽入侵做高亮**（来自 C 方案）：术语、数据、标签用等宽字体 + 薰衣草底色或黑细描边（如 `900+ regression tests`、`OS × AI`），工程师身份感
  - **代码**：JetBrains Mono 类等宽字体

### 3.2 视觉系统汇总（已验收）

组装验收通过（hero-assembled-v3 demo，用户评价「整体很对味」）：

- **色彩**：奶油米底 `#F7F4EE` · 淡薰衣草 `#CDCAE0` · 中紫 `#8A82C9` · 交互紫 `#5E6AD2` · 墨 `#1C1917` · 白卡片
- **字体**：衬线标题（Source Serif/思源宋体）+ 无衬线正文（Inter/思源黑体）+ 等宽高亮 chip（JetBrains Mono）
- **线条语言**：签名元素 = 细框按钮 / 硬投影卡片 / 手绘圈注（悬停重播）；配角 = 动态下划线 / 标尺线 / 角标框
- **动效**：Linear 式级联进场（导航→标语→标题→正文→CTA→卡片，约 0.05–0.78s 阶梯延迟）+ 全站悬停微交互 + 圈注进场/悬停绘制
- **气质边界**：工艺感可以有、华丽发光不要；一切动效服务内容，学术可信度优先

## 4. 技术方案与部署

### 4.1 技术栈：Astro + Markdown/MDX

- **框架**：Astro（静态站点生成器）。选型理由：
  - 内容维护体验契合 §1.5 的要求——**加一篇文章/项目 = 加一个 `.md` 文件**，模板和列表页由构建系统生成
  - 内容集合（content collections）带 frontmatter schema 校验，字段写错构建即报错
  - i18n 路由官方支持，匹配「默认英文 + 中文切换」的双语策略
  - 默认零 JS 出包；本站动效全部是 CSS transition + 少量原生 JS（圈注 `getTotalLength()` 等），不需要 React 运行时
- **内容组织**：`blog/`、`projects/`、`now` 等作为内容集合；固定页面（首页/关于）双语两版，博客单语（见 §2.2）
- **备选已否决**：Next.js（静态个人站过重）；纯手写 HTML（双语 × 多页 × 列表页的重复劳动不可持续）

### 4.2 托管与部署：GitHub Pages

- **链路**：push 源码到 GitHub 仓库 → GitHub Actions 自动 `astro build` → 产物发布到 GitHub Pages
- **日常工作流**：写 `.md` → `git push` → 约 2 分钟后自动上线
- **域名**：先用 `aizyeee.github.io`，不买自定义域名；未来若需要可平滑升级（绑自定义域名 + Cloudflare CDN 解决国内访问速度），源站与内容不动
- **已确认限制均无实质影响**（纯静态托管，无服务器）：
  - 语言自动跳转 → 不需要，默认英文落地，客户端 JS 提示切换
  - 站内搜索 → 首版不做；将来可用 Pagefind（构建时索引 + 浏览器端搜索）
  - 评论 → 首版不做；将来可用 giscus（GitHub Discussions）
  - 访问统计 → 首版不做；将来可接 GoatCounter 等轻量客户端统计

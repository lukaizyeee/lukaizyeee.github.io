# 个人主页 & 简历网站 —— 设计 Spec

> 状态：待用户审阅
> 来源：`discussion.md`（2026-07-03 讨论定稿）
> 范围：**整体框架 + 视觉方案 + 页面模板**。具体文案与文章内容不在本 spec 范围内，后续持续填充（见 §9）。

---

## 1. 目标与定位

- **一句话目标**：让访问者在 30 秒内明白——这是一个正在系统性走向 AI 系统 / OS × AI 方向的本科生，做真项目、有工程品味、正在向 IPADS 方向靠拢。
- **定位**：综合门户 + 长期维护的个人展示面。不是求职简历的网页版（正式简历是独立 PDF，网站仅提供下载）。
- **首要受众**：IPADS 方向的老师、学长学姐、未来学术同行与合作者；HR/面试官是次要受众。
- **基调**：允许体现成长轨迹——学习记录、方向思考是正面内容，不伪装成资深作品集。

## 2. 信息架构

### 2.1 站点地图与路由

双语站，**默认英文落地**（无前缀），中文加 `/zh` 前缀：

| 模块 | EN 路由 | ZH 路由 | 语言策略 |
|---|---|---|---|
| 首页 | `/` | `/zh/` | 双语两版 |
| 项目列表 | `/projects` | `/zh/projects` | 双语两版 |
| 项目 case study | `/projects/<slug>` | `/zh/projects/<slug>` | 双语两版 |
| 博客列表 | `/blog` | `/zh/blog` | 页面骨架双语，文章池共享 |
| 博客文章 | `/blog/<slug>`（唯一规范 URL，无 `/zh` 镜像） | — | **单语**：用什么语言写就发什么，页面骨架跟随文章语言 |
| Now | `/now` | `/zh/now` | 双语两版 |
| 关于 & 联系 | `/about` | `/zh/about` | 双语两版；含 PDF 简历下载 |
| 404 | `/404` | — | 单版（英文 + 简短中文提示） |

- 导航栏常驻语言切换（`EN/中`），在对应页面间互跳；博客文章页的切换回到 `/blog` 或 `/zh/blog` 列表。
- 不做浏览器语言自动跳转（静态托管无服务端）；可选：客户端一行 JS 检测到中文环境时显示不打扰的切换提示条。
- 技术术语（RAG、prompt、focused rerun 等）在中英两版中均保留英文。

### 2.2 首版明确不做

- 独立网页版简历页（PDF 下载入口放关于页）
- 兴趣/非技术板块（未来作为博客分类加入）
- 站内搜索、评论、访问统计（升级路径见 §7.3）
- 深色模式（色板已验证黑底可行，未来可加）
- 横向滚动叙事（rauno 式，讨论中欣赏但未采用）

## 3. 页面框架（模板级，内容占位）

### 3.1 首页 `/`

自上而下：

1. **导航**：左 logo（姓名，衬线体），右 Projects / Blog / Now / About + 语言切换
2. **Hero（方向声明型首屏）**：
   - 等宽 kicker 行（身份标签：方向 · 学校 · 年级）
   - 衬线大标题 = 一句方向声明，关键词（如 `OS × AI`）套**手绘圈注**
   - 无衬线正文 2–3 行，硬指标与关键概念用**等宽 chip** 高亮
   - CTA 行：细框按钮（主）+ 下划线链接（次）
3. **精选项目**：2 张描边卡片（合同审查系统、Lighter），一白一薰衣草底，链向 case study
4. **最近文章**：3–5 条博客列表（标题 + 日期 + 语言徽标）
5. **页脚**：联系方式（邮箱为主）、GitHub、备案性质的极简一行

### 3.2 项目列表 `/projects`

- 编号 + 标尺线章节头
- 核心项目（featured）：大号描边卡片，含一句话简介、技术标签（等宽 chip）、指标
- 其余小项目：紧凑卡片（无详情页），可外链 repo

### 3.3 项目 case study `/projects/<slug>`

Markdown 驱动的长文模板，六段骨架（标题为模板内置章节导航）：

1. 背景与动机 → 2. 系统架构（架构图位）→ 3. 关键设计决策 → 4. 数据与成果（指标用等宽 chip 排列）→ 5. 截图/演示 → 6. 反思与下一步

- 顶部 metadata 条：角色、时间段、技术栈标签
- 长文页启用**阅读进度指示**（细墨线，呼应 rauno 游标偏好）

### 3.4 博客 `/blog` 与文章页

- 列表：时间倒序，每条 = 标题 + 日期 + 标签 + 语言徽标（EN/中）；首版不做分类页，仅简单标签
- 文章页：单栏长文排版，标题衬线、正文无衬线、代码块 JetBrains Mono；阅读进度指示同上

### 3.5 Now `/now`

- 单屏轻页面：**四角括号框**包裹的「当前状态」卡片（在读 / 在做 / 在想三小节）+ 最后更新日期（等宽）
- 内容为手动编辑的 Markdown

### 3.6 关于 `/about`

- 照片可选位 + 完整自我介绍（Markdown）
- 联系方式块 + **PDF 简历下载**（细框按钮）
- 教育背景 / 技能清单（等宽 chip 分组）

## 4. 设计系统

### 4.1 色彩（5 色相封顶）

| 角色 | 色值 | 用途 |
|---|---|---|
| 底色 | `#F7F4EE` 奶油米 | 全站页面底 |
| 表面色 | `#CDCAE0` 淡薰衣草 | hero 大色块、次级卡片底、chip 底 |
| 次级强调 | `#8A82C9` 中紫 | 插画/封面色块、辅助点缀 |
| 交互色 | `#5E6AD2` 蓝紫 | 链接、按钮投影、kicker、高亮标签 |
| 墨色 | `#1C1917` | 正文、标题、线条系统 |
| 卡片 | `#FFFFFF` | 白卡片 |

- 中性阶（次级文字 `#44403C`/`#78716C`、边框 `#E7E2D8` 等）从墨色与米色派生，**不引入新色相**；功能色未来按需补充。
- 强调不完全依赖颜色：黑色线条系统承担结构与强调（§4.3）。

### 4.2 字体

| 用途 | 英文 | 中文 | 加载策略 |
|---|---|---|---|
| 标题 | Source Serif 4（或同类衬线） | 宋体系（Songti SC / Noto Serif CJK 回退栈） | 英文 webfont；**中文不加载 webfont，用系统字体栈**（CJK 字体体积不可接受） |
| 正文 | Inter | 黑体系（PingFang SC / Noto Sans CJK 回退栈） | 同上 |
| 等宽 | JetBrains Mono | — | webfont，subset |

- 等宽字体「入侵」正文做高亮：术语/数据/标签 = 等宽 + 薰衣草底 chip 或黑细描边 chip 两种样式。
- 英文 webfont 做 subset + `font-display: swap`。

### 4.3 线条语言（ink-line 系统）

**签名元素**（形成辨识度，克制使用于关键位置）：

| # | 元素 | 行为 |
|---|---|---|
| 1 | 细框按钮 | 1.5px 墨描边；悬停填充墨色 + 上移 2px + 露出交互紫硬投影 |
| 3 | 描边卡片 | 白/薰衣草底 + 1.5px 墨描边；悬停向左上错位 3px + 右下实心墨色硬投影 |
| 5 | 手绘圈注 | SVG 路径描边动画圈住关键词；**进场自动画一次（级联后约 1.15s）+ 每次悬停重画**；笔色暂定墨黑 |

**实用配角**（全站基础设施）：

| # | 元素 | 用途 |
|---|---|---|
| 2 | 动态下划线 | 导航与正文链接（左入右出） |
| 4 | 编号 + 标尺线 | 章节标题（滚动入视口时线条从 1/4 延展到全宽） |
| 6 | 四角括号框 | Now 卡片、数据高亮（悬停四角合拢成框） |

**圈注实现约束**（讨论中踩坑，必须遵守）：`stroke-dasharray/-offset` 运行时用 `getTotalLength()` 精确设置；CSS 保留偏大兜底值防脚本前闪现；硬编码估算长度会泄漏笔画。

### 4.4 动效系统

- **进场**：Linear 式级联浮现（透明度 + 上移 22px，`cubic-bezier(.21,.6,.35,1)`，0.05–0.78s 阶梯延迟，顺序：导航→kicker→标题→正文→CTA→卡片），圈注随后起笔
- **滚动**：章节标尺线、非首屏模块用 IntersectionObserver 触发一次性 reveal；克制，不做视差
- **悬停**：所有可交互元素必须有反馈（补 rauno 的短板）——按钮/卡片/链接/圈注各按 §4.3 定义
- **长文页**：顶部细墨线阅读进度条
- **边界**：华丽发光效果（Reflect 式）明确排除；动效服务内容，学术可信度优先
- **可访问性**：尊重 `prefers-reduced-motion`——降级为无动画直接显示

### 4.5 组件清单（实现产物）

`InkButton` / `OffsetCard` / `CircleAnnotation`（含悬停重播逻辑）/ `NavLink`（动态下划线）/ `SectionRule`（编号+标尺线）/ `CornerFrame` / `MonoChip`（薰衣草底、描边两种变体）/ `LangSwitch` / `CascadeGroup`（级联进场容器）/ `ReadingProgress` / `PostList` / `ProjectCard` / `BaseLayout`（含 SEO meta、OG）

## 5. 技术架构

### 5.1 技术栈

- **Astro**（静态站点生成）+ Markdown/MDX 内容集合；动效用 CSS + 原生 JS（无 React 等运行时）
- 备选已否决：Next.js（过重）、纯手写 HTML（双语 × 列表页不可持续）

### 5.2 内容组织

```
src/content/
  blog/          # 单语文章池：<slug>.md，frontmatter.lang 标记语言
  projects/
    en/<slug>.md
    zh/<slug>.md # 同 slug 双语成对
  now/
    en.md, zh.md
```

frontmatter schema（Astro content collections 强校验）：

- **blog**：`title, date, lang('en'|'zh'), tags[], description, draft`
- **projects**：`title, period, role, summary, tech[], metrics[], featured(bool), order, cover?`
- 校验失败 = 构建失败（工程品味：错误左移）

### 5.3 部署

- 仓库：`aizyeee.github.io`（用户站仓库，GitHub 用户名 aizyeee 待确认）
- 链路：push → GitHub Actions 执行 `astro build` → 发布 GitHub Pages
- 日常工作流：**写 `.md` → `git push` → 约 2 分钟自动上线**
- 域名：暂用 `aizyeee.github.io`；升级路径 = 自定义域名 + Cloudflare CDN（国内提速），源站不动

### 5.4 站点基建（首版包含）

- SEO 基础：per-page meta、OG 卡片、`sitemap.xml`、`robots.txt`
- 博客 RSS（`@astrojs/rss`）
- favicon：墨线风格字母 monogram（实现阶段设计）

## 6. 验收标准（框架完成的定义）

1. 全部 8 类页面模板可渲染（占位内容），路由与双语切换按 §2.1 工作
2. §4.5 组件全部实现且行为符合 §4.3/§4.4 定义；圈注无「预泄漏笔画」缺陷
3. 级联进场、悬停反馈、滚动 reveal、阅读进度全部生效；`prefers-reduced-motion` 降级正确
4. 加一篇博客 = 新增一个 `.md`，列表/RSS/首页最近文章自动更新，无需改代码
5. frontmatter 错误能让构建失败并给出可读报错
6. GitHub Actions 管线绿灯，站点在 `aizyeee.github.io` 可访问
7. 无 CJK webfont；Lighthouse 性能分在移动端 ≥ 90（静态站应轻松达标）

## 7. 非目标与延后项

## 8. 风险与对策

| 风险 | 对策 |
|---|---|
| 双语固定页维护成本被低估 | 固定页数量被刻意压到 5 类；case study 虽长但更新频率低；博客（高频区）单语 |
| `github.io` 国内访问偶发缓慢 | 已知且接受；升级路径平滑（§5.3） |
| 动效堆叠导致「吵」 | 签名元素只在关键位置出现；配角元素低调；验收含 reduced-motion |
| 内容长期不填充，站点空壳 | 框架验收用占位内容即可上线雏形；Now 页设计为最低成本更新入口 |
